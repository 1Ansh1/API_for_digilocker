PKCE Security

Proof Key for Code Exchange (PKCE) prevents authorization code interception attacks. A temporary code_verifier is cached locally in Redis, and its SHA-256 code_challenge is sent during authorization. Only the party holding the verifier can exchange the returned code.

OIDC Validation

Upon receiving the callback, the backend validates OIDC ID token headers. It dynamically fetches DigiLocker's public keys via its JWKS (JSON Web Key Set) endpoint (cached in Redis), verifies the signature cryptographically, checks for replay attacks via the nonce, and validates the iss (issuer) and aud (audience) claims.

Data Retention

In compliance with privacy policies, cleartext Personally Identifiable Information (PII) is isolated in a separate verification_results table. This data is temporarily kept for verification review, but is permanently pruned using background tasks after 5 minutes. The verifications log retains only non-identifying metrics, hashes, and error status records.

Component Breakdown

FastAPI Service Serves API routers (v1 endpoints), runs background tasks, handles template injection, and orchestrates services via dependency injection.
Redis Cache Implements rate limiting sliding windows, acts as the temporary session store for PKCE parameters, and maintains concurrency locks to prevent double-initiations.
PostgreSQL DB Provides persistent storage for core session states, append-only logs of auditable system actions (audit_events), and temporary demographic details (verification_results).
